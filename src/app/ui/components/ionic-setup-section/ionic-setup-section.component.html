z-control Ionic Setup App
What the app does

The z-control Ionic Setup app is a production-ready starter template for building modern Ionic applications with Angular, Capacitor, and Firebase.

It provides a clean tab-based user interface, built-in quota management, help and support pages, settings management, and a complete Firebase backend integration to help you accelerate development of your own apps.

✨ Key Features

  • Quota Management: Monitor API usage with configurable limits, warnings, and automatic monthly resets.
  • Modern Navigation: Clean tab-based interface with dedicated areas for features, settings, and support.
  • Built-in Help System: Integrated documentation, FAQs, and user guidance.
  • Settings Framework: Ready-to-use settings pages with support for configuration, feedback, changelogs, privacy policies, and application information.
  • Privacy-Focused Design: User input is processed by external services without being stored by the application.
  • Cross-Platform Deployment: Run as a web application or deploy natively to Android using Capacitor.

🛠️ Tech Stack

  • Framework: Ionic 8 + Angular 20
  • Language: TypeScript
  • Styling: SCSS and Ionic CSS Variables
  • Build Tools: Angular CLI
  • Icons: Ionicons and z-control Icons
  • State Management: RxJS
  • Backend: Firebase (Firestore, Cloud Functions, Hosting) managed through the z-control Backend Functions project
  • Testing: Karma + Jasmine (frontend), Vitest (backend)
  • Deployment: Firebase Hosting and Capacitor (Android)

The web version of this application is hosted on Google Firebase Hosting.

What are the limitations and monthly quotas?
Why are quotas used?

This Ionic Setup app demonstrates how monthly usage quotas can be managed for features that consume limited or paid resources.

Users are notified well before a quota is reached. Once a monthly quota has been exhausted, the affected feature is temporarily unavailable until the beginning of the next month, when all quotas are automatically reset.

The framework supports two types of quotas:

  • Shared monthly quota: A total usage limit shared by all users of the application.
  • Per-user monthly quota: An individual usage limit that prevents a single user from consuming a disproportionate share of the available resources.

Quota configuration used in this demo:
Shared monthly quota: {{ parameters?.maxFeatureCharsTotalPerMonth | number: "1.0-0" : "en" }} characters for all users.
Per-user monthly quota: {{ parameters?.maxFeatureCharsUserPerMonth | number: "1.0-0" : "en" }} characters per user.

How can I check my quota usage?

On the Feature Page

Your current monthly usage is displayed in the expandable Monthly Quota section at the bottom of the feature page.

If either your personal quota or the shared quota is reached, the main feature (for example, Search Related Words) becomes unavailable until the next monthly reset. All other application features remain available.

On the Statistics Page

The Statistics page provides anonymized usage information, including overall quota consumption and aggregated device statistics across all users.

All statistics are collected and displayed in an aggregated form to help protect user privacy.

Inline Help
Skeleton Help System to easily adapt for your own app

The Help Page is a skeleton that you can easily adapt when building your own app based on this template. It covers all important topics and must be expanded with your specific information, and instructions.

Both the Web App and Android App include this inline help accessible by clicking the ? icon in the top right corner.

The help includes a table of contents with jump-to-section navigation for quick access to specific topics.

The tutorial is specific to the version you are using. Information relevant to mobile apps is omitted in the web-app help, and web-app specific guidance is omitted in the mobile-app help.

Help Topics Covered:
  1. What Can the App Do
  2. How to Use the App
  3. Web Version versus Mobile Version
  4. Technical Information
  5. Troubleshooting
