z-control Backend Functions
Shared backend code across z-control apps

z-control Backend Functions is the shared backend repository for z-control applications. It contains the canonical backend logic used across multiple apps, including Firebase Cloud Functions, Firestore utilities, and related backend helpers.

This repository keeps shared backend code consistent across all frontend applications while allowing each app to remain focused on its own frontend-specific features.

Frontend-specific changes belong in the individual app repositories. Shared backend code should be implemented and deployed only from this repository.

Why this matters

  • Single source of truth for shared backend logic.
  • Consistent behavior across all z-control apps.
  • Clear separation between frontend and backend responsibilities.
  • Safer and more predictable deployments.
  • Centralized z-control Github Analytics functions.

Key Features
  • Single source of truth for shared backend code across all z-control apps.
  • Firebase Cloud Functions for API handling, data processing, and Firebase integration.
  • Quota management to control API usage and help avoid unexpected costs.
  • Unit tests with Vitest and coverage reporting.
  • Documentation for setup, usage, deployment, and development workflows.
How to Use
Development workflow
When you need backend changes for a frontend app:
  1. Implement the shared backend change in this repository.
  2. Run and verify the backend locally with Firebase emulators.
  3. Connect the target frontend app to the emulator backend.
  4. Test the frontend/backend integration.
  5. Deploy Firebase Functions from this repository only.
  6. Deploy the frontend app separately after backend validation.
GitHub Analytics for z-control Repositories
We track GitHub repository activity and contributions to better understand engagement and development trends across z-control projects.

GitHub's Insights > Traffic section provides useful metrics, but only for the last 14 days. To keep a longer-term record, Firebase Functions periodically collect these metrics and store them in Firestore. This lets us analyze GitHub activity over time instead of being limited to GitHub's built-in reporting window.
