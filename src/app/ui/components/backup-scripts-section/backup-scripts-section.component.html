z-control Backup Scripts
Automate Backups, Cleanup, and Disk Usage Logging

z-control Backup Scripts is a collection of open-source PowerShell and batch scripts for Windows, designed to help you:

  • Back up development projects (with exclusions for build/dependency folders)
  • Back up non-committed files (such as .env, .vscode, etc.)
  • Clean up dev folders (remove node_modules, dist, etc. recursively)
  • Log disk/folder usage (CSV reports for auditing and cleanup)

Each script is documented and can be run interactively or scheduled for automation. All scripts are compatible with PowerShell 5 and 7.

Typical use cases:

  • Automated project backups (with logs)
  • Quickly reclaim disk space in dev folders
  • Audit disk usage for large folders
  • Back up important files before deleting or migrating projects

Key Features
  • Recursively back up project folders and files
  • Exclude common build and dependency folders (customizable)
  • Log processed items to CSV (folders only or folders and files)
  • Example scripts for non-committed files
  • Clean up dev projects by deleting large or unnecessary folders (e.g., node_modules, www, dist, .angular)
  • Log all deleted folders and (optionally) files to CSV for audit
  • Ready for scheduled automation via batch files (for both backup and cleanup)
  • Log disk space usage for any disk drive or directory, with flexible depth and output formats (CSV, Excel-ready)
  • Example logs and Excel files for disk usage analysis
 How to Use
Getting started is easy:
  1. Download scripts from GitHub or your project repository.
  2. Review the included markdown guides for each script type.
  3. Customize parameters as needed (source, destination, log file, etc.).
  4. Run interactively or schedule via Windows Task Scheduler.
  5. View logs and CSV reports for results and auditing.

For step-by-step instructions, open the relevant documentation using the buttons above.

Automation with Windows Task Scheduler
  • You can use all batch scripts in a Windows Scheduled Task to automate backups.
  • Set the task to run the batch file at your preferred schedule.
  • Ensure the user account running the task has permission to access the source and backup directories.
